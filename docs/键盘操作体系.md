# 项目键盘操作体系说明

本项目支持高度可配置的游戏键盘操作，所有热键均可通过配置自定义，支持多场景（通用、工业、战斗）下的快捷键映射。本文档详细介绍键盘操作的设计、默认键位、扩展方式及开发注意事项。

---

## 1. 按键配置体系概览

- **热键统一管理**：所有键盘操作均通过 `KeyConfigManager` 管理，支持用户自定义和默认值。
- **三大类热键**：
  - 通用（common）：如地图、交互、背包等
  - 集成工业（industry）：如工业规划、放置传送带等
  - 战斗（combat）：如连携技等
- **国际化支持**：所有键名均有 i18n 支持，便于多语言适配。

## 3. 按键调用方式

### 3.1 业务层接口

- `press_game_key(key, key_type='common', ...)`：发送指定类型的热键，自动适配用户自定义。
- `press_key(key, **kwargs)`：发送通用按键（如地图、交互等）。
- `press_industry_key(key, **kwargs)`：发送工业专用热键。
- `press_combat_key(key, **kwargs)`：发送战斗专用热键。

### 3.2 按键映射流程

1. 业务层调用如 `press_key('m')`。
2. 通过 `KeyConfigManager.resolve_common_key('m')` 查找逻辑键名对应的配置键名。
3. 若用户有自定义，则取自定义值，否则用默认值。
4. 最终通过底层 `send_key`进行物理按键模拟。

---

## 4. 用户自定义与国际化

- 用户可通过配置文件自定义所有热键。
- 支持多语言键名，详见 `i18n/` 目录下各语言包。

---

## 5. 开发注意事项

- **严禁使用send_key**，必须通过 `press_key` `press_industry_key` `press_combat_key` 调用。
- 若热键可自定义，则不可用作模板图片（否则模板匹配会因用户自定义失效）。
- 详细开发规范见 `docs/dev/DEVELOPMENT.md`。
